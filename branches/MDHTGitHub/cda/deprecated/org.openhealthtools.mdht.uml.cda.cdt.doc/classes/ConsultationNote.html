Consultation Note

[ClinicalDocument: templateId 2.16.840.1.113883.10.20.4]

  1. SHALL conform to General Header Constraints template (templateId: 2.16.840.1.113883.10.20.3) (CONF-CO-1)
  2. SHALL contain [1..1] code, which SHALL be selected from ValueSet ConsultDocumentTypeCodes DYNAMIC (CONF-CO-2)
  3. SHALL contain [1..1] component, such that it
    1. contains History Of Present Illness (templateId: 1.3.6.1.4.1.19376.1.5.3.1.3.4)
    (CONF-CO-27)
  4. SHALL contain [1..1] component, such that it
    1. contains Physical Examination Section (templateId: 2.16.840.1.113883.10.20.2.10)
    (CONF-CO-28)
  5. MAY contain [0..1] component, such that it
    1. contains CCD Problem Section (templateId: 2.16.840.1.113883.10.20.1.11)
    (CONF-CO-32)
  6. MAY contain [0..1] component, such that it
    1. contains CCD Procedures Section (templateId: 2.16.840.1.113883.10.20.1.12)
    (CONF-CO-33)
  7. MAY contain [0..1] component, such that it
    1. contains Past Medical History Section Consult (templateId: 2.16.840.1.113883.10.20.4.9)
    (CONF-CO-34)
  8. MAY contain [0..1] component, such that it
    1. contains CCD Immunizations Section (templateId: 2.16.840.1.113883.10.20.1.6)
    (CONF-CO-35)
  9. MAY contain [0..1] component, such that it
    1. contains CCD Medications Section (templateId: 2.16.840.1.113883.10.20.1.8)
    (CONF-CO-36)
  10. MAY contain [0..1] component, such that it
    1. contains CCD Alerts Section (templateId: 2.16.840.1.113883.10.20.1.2)
    (CONF-CO-37)
  11. MAY contain [0..1] component, such that it
    1. contains CCD Social History Section (templateId: 2.16.840.1.113883.10.20.1.15)
    (CONF-CO-38)
  12. MAY contain [0..1] component, such that it
    1. contains CCD Family History Section (templateId: 2.16.840.1.113883.10.20.1.4)
    (CONF-CO-39)
  13. MAY contain [0..1] component, such that it
    1. contains Review Of Systems Section (templateId: 2.16.840.1.113883.10.20.4.10)
  14. MAY contain [0..1] component, such that it
    1. contains CCD Vital Signs Section (templateId: 2.16.840.1.113883.10.20.1.16)
    (CONF-CO-42)
  15. MAY contain [0..1] component, such that it
    1. contains General Status Section (templateId: 2.16.840.1.113883.10.20.2.5)
    (CONF-CO-43)
  16. MAY contain [0..1] component, such that it
    1. contains Diagnostic Findings
    (CONF-CO-44)
  17. MAY satisfy: Implementations MAY use local codes in translation elements to specify a local code that is equivalent to the document type. An example of this is shown below. (CONF-CO-3)
  18. SHALL satisfy: If pre-coordinated document type codes are used, values used in the assignedAuthor/code and assignedAuthor/author/functionCode elements SHALL NOT conflict with ClinicalDocument/code. (CONF-CO-4)
  19. SHALL satisfy: If pre-coordinated document type codes are used, values used in encompassingEncounter/location/healthCareFacility/code SHALL NOT conflict with ClinicalDocument/code. (CONF-CO-5)
  20. SHALL satisfy: A Consultation Note contains either ReasonForReferral or ReasonForVisit, but not both. (CONF-CO-26)

   xor self.getSections()->exists(section : cda::Section | not section.oclIsUndefined() and section.oclIsKindOf(cdt::ReasonForVisitSectionConsult))
