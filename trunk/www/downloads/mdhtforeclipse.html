Model-Driven Health Tools (MDHT)

Eclipse Update and Archive Site Installation Instructions

All project builds are intended to be installed in your Eclipse platform using the Eclipse Update Manager, accessed using the

Help > Software Updates... menu for Eclipse Version 3.4 and RSM. menu image

...or...

Help > Install New Software... menu for Eclipse Version 3.5. menu image

The update site is currently located on a SourceForge server where we have access to web file directories (unavailable on the OHT server). The update sites contain release and milestone builds, but not integration development builds. If you are unfamiliar with use of the update manager:

  1. Add an update site location (Eclipse 3.4 screen image or Eclipse 3.5 screen image) using the following MDHT Update Site Location

    http://oht-modeling.sourceforge.net/updates

    or if you downloaded the site locally, use the Add Archive site option and browse to where you saved the archived site.

  2. Select the appropriate features based on your UML Tooling Configuration and Workspace Selection from the following options (Eclipse 3.4 features image or Eclipse 3.5 features image)
    • CDA Options
      • CDA for Simple UML Tooling
      • CDA for RSM UML Tooling
      • CDA for Papyrus UML Tooling
    • RIM Options
      • RIM for Simple UML Tooling
      • RIM for RSM UML Tooling
      • RIM for Papyrus UML Tooling
  3. Select, Install (Eclipse 3.4) or Finish (Eclipse 3.5), Accept and Install All, Restart.
