Model-Driven Health Tools (MDHT) Downloads

All project builds are intended to be installed in your Eclipse platform using the Eclipse Update Manager, accessed using the Help > Software Updates... menu. The update sites are currently located on a SourceForge server where we have access to web file directories (unavailable on the OHT server). The update sites contain release and milestone builds, but not integration development builds. If you are unfamiliar with use of the update manager:

  1. Add an update site location (screen image)
  2. Select features for installation (screen image)

Install from one or both update sites URLs:

  • For CDA Tools: http://oht-modeling.sourceforge.net/cda/updates
  • For HDF Tools: http://oht-modeling.sourceforge.net/updates>

A ZIP of the update site is created for each build, which you can download and configure as a local update site location in the Eclipse update manager. This is useful for organizations that prohibit use of external update sites via their firewall.

Milestone Builds

Milestone builds represent planned iteration points in our development process. User testing and feedback is solicited to validate readiness for promotion to release status. Milestone builds may contain pre-release software that is pending approval for redistribution under the Open Health Tools Software User Agreement.

Recommended procedure is to install using the update site. Instructions at the top of this page.

CDA Tools

Milestone M1 6th October 2009 (update site ZIP)

IMPORTANT NOTE: If you have ANY test build installed prior to 1st July 2009, then you MUST uninstall it prior to using the update site to install this release candidate. The Eclipse feature IDs have changed. If you do not uninstall the prior features, then you will have duplicate functionality where all menus, properties tabs, etc. will appear twice.
