Windows Download

The Windows Downloads is an archived version of the completely configured Eclipse UML Tooling Installation for Windows. The files themselves can be rather large but represent the easiest option for installing MDHT.

Update Manager

All project builds are intended to be installed in your Eclipse platform using the Eclipse Update Manager, accessed using the Help > Software Updates... menu in eclipse 3.4 or Help > Install New Software... menu in eclipse 3.5 . The update sites are currently located on a SourceForge server where we have access to web file directories (unavailable on the OHT server). The update sites contain release and milestone builds, but not integration development builds.

Archived Update Site

Under some circumstances with firewall and other security constraints, it will be necessary to download an archived version of the Eclipse Update Site. The installation process is generally the same as the Update Manager Installation process but instead of installing from a remote site you install from a local instance. The disadvantage of this approach is updates to the plugins will not be managed automatically by the eclipse platform. Organizations or individuals using this option will have to download updates to keep software updated.

Custom UML Tooling Installation

Custom UML Tooling Installations document the process that the Windows archives were created with. You can use this method if you want to apply more discretion on what is installed. The approach is the most complicated option and requires a good deal of knowledge of Eclipse and Eclipse Plugins to complete successfully.

MDHT Workspace Configurations

MDHT Workspace Options

The following configurations will allow you to download and open the projects and models associated with the respective tool.

Project Set File

Project set files allow you to quickly import projects from Subversion into your workspace. This approach requires you have Subversion install but is the simplest approach to getting and maintaining the latest projects from MDHT. Here are instructions to install Subversion
You have to download the PSF file on to your local disk, and then you can import projects in Eclipse using the Team Project Set Wizard by selecting File->Import->Team->Team Project Set and following the steps to import a PSF.

Archived Project Workspace

Archived Project Workspace will contain a completely configured Eclipse Workspace including all the necessary projects and plugin configurations from the time of the release of the MDHT project. These projects will be detached from the MDHT Subversion site. This option is currently the best option for Simple and Papyrus tool configurations to get started quickly and without having too configure the workspace.

You have to download the Workspace Archive and extract the contents to a location on your computer before starting Eclipse. On start up, Browse using Select Workspace dialog (Screen Shot) and select the location of the extracted workspace. The extracted workspace is considered the directory containing the projects.

Subversion Repository

The Subversion Repository option requires the user to understand and use the Subversion plugin. It is recomended for experienced eclipse users who know and understand how to switch perspectives and potentiall add views. Here are instructions to install Subversion

CDA Tools - Tools supporting UML based Clinical Document Architecture

The CDA Tools configuration includes a series of Eclipse UML Model Projects containing CDA based models such as C32 and IHE. The projects also contain an example CDA extension model and test code.
