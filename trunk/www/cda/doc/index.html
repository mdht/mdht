Getting Started Guides

Getting Started with Installation and Configuration

First step is to properly install the correct platform and configure the MDHT workspace. The MDHT installation process has two distinct parts, a platform installation and a workspace configuration. You must complete both steps completely to properly utilize the MDHT tools. Please follow the instructions on the Platform Installation Page. When you are done, you will have the platform installed as well the MDHT workspace properly configured.
The following guide will walk you through the steps required to select and install the correct platform. You will be taken to the installation page where will learn how to install the MDHT tool set.
  • Getting Started with Platform Installation
The following guide will walk you through the steps required to configure a workspace for use with the MDHT tools. You will be taken to the installation page where you will learn how to configure your own work space.
  • Getting Started with Workspace Configuration

Getting Started with UML...
Quickly get started using UML models. This guide features the use of the Easy UML Editor which is available on all platforms.
  • Getting Started with Easy UML Editor
Quickly get started using the Rational Software Modeler. This guide features the use of the RSM Workspace and Modeling Perspective. Only use this guide if you have installed RSM from IBM.
  • Getting Started with RSM
Quickly get started using the Eclipse UML Project Papyrus. This guide features the use of the Papyrus Workspace and Modeling Perspective. Only use this guide if you have installed Papyrus from MDHT.
  • Getting Started with Papyrus

Getting Started with CDA...
Learn how you can create your own CDA based model. The guide features the steps required to create a new CDA project where you can extend existing CDA models such as HITSP, IHE, and CCD.
  • Getting Started with CDA Modeling
Learn how you can generate actual Java source code from your a CDA based model. This guide features the steps required to generate, compile and run Java based CDA which will create and validate an actual CDA document instance.
  • Getting Started with CDA Java Code Generation
