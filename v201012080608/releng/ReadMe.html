Release Engineering Guide - Deprecated

This document describes the process for using the automated build for this project. It uses the Eclipse Release engineering framework (releng) to perform this task.

Basic steps

  1. Fetch the eclipse base builder from eclipse CVS repository
  2. Download and unzip the subversion pde build plugin
  3. Fetch the openhealthtools-pde-build builder from SVN
  4. Start build

Build Instructions

  1. Create a new folder for the build (e.g. releng) and CD to this directory.
  2. Fetch the eclipse base builder from eclipse CVS repository

    3. Fetch the eclipse base builder using the following command

    cvs -d :pserver:anonymous@dev.eclipse.org:/cvsroot/eclipse export -r HEAD org.eclipse.releng.basebuilder
  3. Download the subversion pde build plugin from the sourceforge.net svn-pde-build project

    4. Download the svn-pde-build zip file from the sourceforge web site.

    Extract the svn-pde-build zip file over the org.eclipse.releng.basebuilder directory.

  4. Fetch the svn-pde-build builder from SVN

    5. svn export https://mdht.projects.openhealthtools.org/svn/mdht/trunk/releng
  5. Start build

    6. CD to the builder directory created by the svn export and start the build with the following command.

    java -cp ../org.eclipse.releng.basebuilder/plugins/org.eclipse.equinox.launcher.jar org.eclipse.core.launcher.Main -application org.eclipse.ant.core.antRunner -buildfile bui ld.xml -Dcomponent=openhealthtools-mdht-pde-build

    The results of the build will be in the directory ...releng\mdht
